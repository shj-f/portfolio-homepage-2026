피그마 Make AI에 넣을 수 있게 원페이지 구성 기준으로 정리했어요.

개발자 포트폴리오 웹사이트를 제작해주세요.

전체 분위기는 일반 기업 홈페이지보다 “개발자 개인 블로그 + 포트폴리오”가 결합된 형태로 구성합니다.
과한 그래픽이나 화려한 3D 요소보다는, 깔끔한 타이포그래피와 충분한 여백, 프로젝트 이미지와 기술 기록이 잘 보이는 구조를 우선합니다.

디자인 스타일은 미니멀하고 세련된 개발자 포트폴리오 스타일입니다.

* 밝은 배경 또는 아주 연한 그레이 배경
* 포인트 컬러는 블루 또는 딥 네이비 계열
* 텍스트 가독성이 높은 큰 타이포그래피
* 카드형 프로젝트 레이아웃
* 코드, 태그, 기술 키워드를 활용한 개발자 느낌
* 너무 스타트업 랜딩페이지처럼 과장된 그래픽은 지양
* 블로그처럼 읽히되 포트폴리오로서 신뢰감이 느껴지는 디자인
* Desktop 기준으로 디자인하되 모바일 반응형을 고려한 구조

페이지는 스크롤 기반의 One Page Portfolio로 구성합니다.

상단에는 고정 헤더를 배치합니다.

5. WORK LOG SECTION

Work Log는 블로그 형식의 문제 해결 기록을 보여주는 섹션입니다.

Projects와 역할을 구분합니다.

Projects는 결과물 중심,
Work Log는 문제 해결 과정과 기술적인 판단 중심입니다.

섹션 타이틀:

Work Log

서브 문구:

작업 과정에서 마주한 문제를 분석하고,
재사용 가능한 해결 방식으로 기록합니다.

Work Log 카드 구성:

* 카테고리
* 제목
* 짧은 설명
* 날짜 또는 태그
* Read More 버튼 또는 화살표 아이콘

카테고리 예시:

Responsive
PHP / GnuBoard
Server
SMTP
SSL
Performance
Interactive UI
Troubleshooting

Work Log에는 공통 기술 문제 해결 경험을 넣습니다.

예시:

Gmail SMTP 메일 발송 오류를 점검하는 순서
SSL 적용 후 리다이렉트 루프 해결 방법
PHP 환경에서 이미지 업로드 오류가 발생했을 때 확인할 항목
Swiper 두 개를 하나의 네비게이션으로 연동하는 방법
반응형 웹에서 콘텐츠 우선순위를 정하는 기준

특정 프로젝트에서 발생한 문제를 Work Log에 넣는 것도 가능합니다.

이 경우 제목 앞에 프로젝트명을 명시할 수 있습니다.

예시:

[B Project] 가로형 전자카탈로그를 모바일용 구조로 재구성한 과정
[C Project] model-viewer에서 제품 중심 회전이 어색할 때 해결한 방법
[A Project] 디자인 시안과 실제 반응형 구현 간 차이를 줄인 방법

단, Work Log는 프로젝트 전체 소개가 아니라
특정 문제 하나를 해결한 기록으로 구성합니다.

Projects와 Work Log의 관계는 다음과 같습니다.

Projects:
결과물, 담당 범위, 기술 스택, 화면, 프로젝트 소개

Work Log:
문제 상황, 원인 분석, 해결 방향, 구현 방식, 결과, 재사용 가능한 인사이트

Work Log는 블로그 리스트처럼 세로형 카드 또는 가로형 리스트로 구성합니다.
각 카드에는 작은 태그와 날짜가 들어가면 좋습니다.

전체 페이지 UX 방향

* 섹션 간 이동은 부드러운 스크롤
* 헤더 메뉴 클릭 시 해당 섹션으로 앵커 이동
* 현재 보고 있는 섹션 메뉴 활성화
* 프로젝트 카드는 hover 시 이미지 확대 또는 정보 오버레이
* Work Log 카드는 hover 시 화살표 이동 또는 배경 변화
* 텍스트가 너무 작지 않게 구성
* 원페이지지만 각 섹션이 독립적인 블로그 글처럼 읽히도록 구성
* 프로젝트는 시각 중심
* Work Log는 텍스트와 문제 해결 구조 중심
* 전체적으로 신뢰감 있고 실무적인 개발자 포트폴리오 느낌으로 제작
